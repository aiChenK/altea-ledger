# 🐉 阿尔特里亚大陆考勤簿 (altea-ledger)

这是一个专为龙之谷（Dragon Nest）游戏玩家打造的**高密度、暗黑磨砂玻璃风格 (Glassmorphism)** 的角色周常与日常状态记录管理系统。它完全取代了繁琐难用的传统 Excel 表格，提供更直观的多角色对比、自动周期重置、在线配置管理及周常历史归档备份功能。

---

## ✨ 核心特性

- 🎮 **多角色横向极窄对比**：针对 1080p 及不同分辨率进行了专门的高密度优化，角色列宽锁定在 `76px`，支持一屏快速横向对比。
- ⚡ **智能重置机制**：
  - **日常重置**：每天早上 **09:00** 自动重置（签到、喝药、每日、幸运、挂机阶段等状态）。
  - **周常重置**：每周六早上 **09:00** 自动重置周关卡。
  - **自动触发**：后端在每次请求时会自动根据当前标准时间对比上次重置时间戳，自动进行静默重置与数据校准。
- 🦢 **资产与 Buff 管理**：
  - 记录各角色的金币、时装套数等资产。
  - 黄金鹅 Buff 到期时间可视化选择，支持加快捷天数（+7天）以及精确倒计时提醒。
- ⏳ **历史周常归档**：
  - 在每周重置（自动或手动强制重置）时，系统自动对当周的所有状态及备忘生成**历史快照**。
  - 历史归档上限可动态配置（默认保留最近 50 个版本），超出后自动剔除最早的存档。
  - 提供侧边归档抽屉和只读大表格详情弹窗，并提供一键清空归档的安全选项。
- 📌 **流式备忘贴纸**：
  - 全局备忘录采用便签标签贴纸（Tag）设计，回车一键生成标签，点击标签尾部 `×` 随时删除。
  - 历史快照大弹窗中也会同步呈现归档时的只读备忘贴纸。
- ⚙️ **在线可视化配置**：
  - 允许在网页上直接新增或删除角色、新增或删除每周关卡，并支持自由微调各关卡的基础通关次数（如：绿龙硬核可通关 1 次，守卫者可通关 2 次等）。
  - 支持直接在前端配置修改每日、每周重置点及历史记录上限。
- 🔒 **安全访问验证**：
  - **默认免密**：未设置密码时直接进入系统。
  - **环境变量注入**：支持通过 `ADMIN_PASSWORD` 环境变量设定访问密码，支持 Docker。
  - **系统面板修改**：可以在网页端配置管理面板中修改或清空访问密码。
  - **浏览器记住密码**：输入正确密码后，浏览器将通过 `LocalStorage` 记住，避免在同一设备上重复输入。

---

## 🛠️ 技术栈

- **前端 (Frontend)**：React (Vite) + Hooks (`useState`, `useEffect`, `useRef`) + Vanilla CSS (Glassmorphism 视觉系统)
- **后端 (Backend)**：Node.js + Express + CORS
- **本地存储 (Database)**：轻量级本地 JSON 文件存储，数据实时写入与同步。

## 🚀 快速开始

### 1. 安装依赖

在项目根目录下执行：

```bash
npm install
```

### 2. 本地开发模式

在本地同时启动前端 Vite 开发服务器和后端 Express API 服务器：

- 启动前端开发服务器（支持热重载，默认运行在 `5173` 端口，已自动配置 `/api` 代理到后端）：
  ```bash
  npm run dev
  ```
- 启动后端 API 服务（默认运行在 `3001` 端口）：
  ```bash
  npm run server
  ```

### 3. 生产环境构建与启动（一键运行）

如果您想直接将程序编译并托管运行在服务器或本地局域网，可直接运行：

```bash
npm start
```
> 该命令会先执行 `npm run build` 打包前端，然后由后端 `3001` 端口直接托管静态包 and API 服务。
> 启动后，请在浏览器中直接访问：**[http://localhost:3001](http://localhost:3001)**

### 4. Docker 部署（推荐）

本项目官方镜像已托管在 Docker Hub：**[aichenk/altea-ledger](https://hub.docker.com/r/aichenk/altea-ledger)**。

直接拉取并运行容器（挂载本地数据目录、设置中国标准时区并配置访问密码）：
```bash
docker run -d \
  -p 3001:3001 \
  -v $(pwd)/data:/app/data \
  -e TZ=Asia/Shanghai \
  -e ADMIN_PASSWORD=your_password \
  --name altea-ledger \
  aichenk/altea-ledger:latest
```

> [!IMPORTANT]
> **关于访问密码验证 (`ADMIN_PASSWORD`)**
> - 可在运行容器时传入 `-e ADMIN_PASSWORD=your_password` 环境变量来配置初始访问密码，留空则为免密模式。
> - 进入系统后，您可以在网页端的配置管理面板中修改或清空访问密码，系统修改结果将持久化并覆盖环境变量的设定。
> - 输入正确密码后，浏览器将通过 `LocalStorage` 记住密码，同一设备无需多次输入。
>
> **关于时区设置 (`TZ`)**
> 龙之谷的日常（09:00）和周常（周六 09:00）自动重置逻辑高度依赖容器的本地系统时间。请务必根据您的时区传入 `-e TZ=Asia/Shanghai` 环境变量。
>
> **关于数据持久化**
> 本项目的数据全部存储在 `/app/data` 目录。挂载 `-v $(pwd)/data:/app/data` 可确保在容器重启时，您的角色进度与配置数据不丢失。
> 容器初次运行时，如果宿主机挂载的 `./data` 目录为空，请务必先将项目源码中的 `data/config.json` 拷贝到该宿主机挂载目录下，否则系统将因缺少初始化配置而无法正确展现默认数据。

### 5. Docker Compose 部署（推荐）

如果您拥有 Docker Compose 环境，可以使用项目根目录下的 `docker-compose.yml` 配置文件进行一键部署。

#### 启动服务
只需在项目根目录运行：
```bash
docker compose up -d
```

#### `docker-compose.yml` 配置：
```yaml
version: '3.8'

services:
  altea-ledger:
    image: aichenk/altea-ledger:latest
    container_name: altea-ledger
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data
    environment:
      - TZ=Asia/Shanghai
      - PORT=3001
      - ADMIN_PASSWORD=your_password  # 访问密码，留空表示免密模式。也可直接在系统网页配置面板中修改
    restart: always
```

## ⚙️ 重置规则

- **重置检测**：系统无后台常驻定时器，避免了多进程冲突。每次前端获取数据（`GET /api/status`）或保存数据时，后端均会自动调用 `checkAndReset`，安全快捷地进行跨周期判定。
- **配置同步**：当您在“配置管理面板”添加新角色、关卡或删除旧项目时，系统会自动与 `data.json` 中的数据结构进行差异校准（Align），移除废弃字段并为新字段填充默认初始值，避免脏数据导致页面崩溃。
